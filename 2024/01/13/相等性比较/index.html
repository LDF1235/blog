相等性比较

整理 js 中相等性比较相关的知识点。

宽松相等 ==

使用宽松相等时，如果变量是相同类型，参照严格相等的比较规则，变量是不同类型时将会发生类型转换。

  • 如果其中一个是 Symbol，另一个不是 Symbol，返回 false。
  • undefined == null 返回 true。
  • 如果其中一个是 true 或者 false ，true 转换成 1，false 转换成 0，再进行比较。
  • 如果一个是字符串，一个是数值，把字符串转换成数值再进行比较。
  • 如果其中一个是对象，另一个是数值或者字符串，先尝试使用 valueOf() 方法，再尝试使用 toString() 方法转换为原始值( Date 类执行 toString() 转换)。

严格相等 ===

数组，对象，函数的比较就不说了，就说几个特殊的点。

  • NaN === NaN 返回 false 。
  • +0 === -0 返回 true 。
  • 一些数组方法使用严格相等，例如 IndexOf，lastIndexOf，这些方法不能用来查找 NaN 。
  • switch 语句使用的严格相等。

同值相等 Object.is()

Object.is() 大部分的比较结果和严格相等一致，除了下列特殊情况。

  • Object.is(+0, -0)) 返回 false 。
  • Object.is(0, -0)) 返回 false 。
  • Object.is(NaN, NaN)) 返回 true 。

零值相等

类似 Object.is()，但 +0 和 -0视为相等。
数组的 includes 方法、Map、Set 使用零值相等比较。
